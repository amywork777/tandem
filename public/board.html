🚲 Hop on — open a checklist

tandem is a shared to-do board between you and your AI agents — one markdown file on your machine that everyone reads and writes. You steer, they pedal.

Open files picks your checklist .md files (each becomes a tab up top) · New checklist starts a fresh project. Everything is editable in place, and agent updates appear on their own.

Chromium browsers (Chrome, Edge, Arc). New here? See how it works · try the demo · the rules.

Click any text to edit — saves as you type · Enter in an item adds a new one below · Backspace on an empty item deletes it · Raw = edit the whole file

The rules

Ownership — only humans check boxes or cross items out; settled items are never resurfaced. Agents only append.

Markers

  • → 🤖 Name: in progress — … claim + lock
  • → 🤖 Name: done — … awaiting your check
  • → ❓ Name: … blocked on you
  • ⚠️ HOLD Name topic shared-file lock
  • → 💬 note your feedback — the 💬 button on any item adds one

Who's who — every name is a colored badge. Agents each get their own badge color, the same on items and in the strip up top. The purple 💬 badge is YOU. Red badges (❓ question, ⚠️ HOLD) need your attention. Text after a badge = status: green done (waiting on your check), amber in progress.

You pick the roles — the Rules: line at the top of the checklist says who does what (e.g. roles: Fable = planner · Codex = coder). It's plain text: click it and rewrite it anytime; agents follow it.

Activity dots — agents end each marker with the time (· 14:32) and refresh it while working; the crew strip turns that into ● active now / ● 25m ago / ○ last seen 3h ago. No dot = the file has no stamps yet.

Any number of agents can ride — but names must be unique. Two copies of the same agent get numbered names (Codex-1, Codex-2) in their kickoff lines; each gets its own badge and lane.

Checked items sink to the bottom of their section (display only — the file keeps your order).

Files — only checklist.md is required (any lone .md works). Bigger projects add decisions.md (the why), archive.md (history), and agents.md (point new agents here first).

Write safety — the file is hot: agents re-read before every write and write atomically. Your checkbox states are sacred.
