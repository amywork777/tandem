How the ride works

  1. Dump your notesyou Your AI turns a meeting transcript or a brain-dump into a checklist file on your computer.
  2. Agents work the listagents Every update lands on the item with a colored name badge, so you always see who did what.
  3. You reviewyou Good? Check it off — it sinks to the bottom. Not quite? Hit 💬 and say what to fix; the agent replies on the same item.
  4. Repeat until doneagentsyou Updates appear on the board by themselves, and the whole history stays in one .md file you can open anywhere.
You pick the roles — and the crew size. The Rules: line at the top of every checklist says who does what — e.g. roles: Fable = planner · Codex = coder. It's plain text: rewrite it on the board anytime, and the agents follow it from their next read. Ride with as many agents as you like — running two copies of the same one? Give them unique names (Codex-1, Codex-2) in their kickoff lines, and each gets its own badge and lane.

For your AI

Every project folder gets an agents.md — operating instructions any agent can follow (marker grammar, write safety, what it may and may not touch). Kick each agent off with one line: "you're Codex — read agents.md in this folder and pick up your work." The name is the only thing the file can't tell them; roles, rules, and the work itself they read for themselves. Templates, the full protocol, and a Claude Code skill live in the repo.

Or just paste this into your agent(s)

No setup — one checklist.md plus this block pasted into any agent (Claude, Codex, anything) is a complete tandem. Works as a system prompt, a first message, or the top of an AGENTS.md.

You're joining a "tandem" board: a checklist in ONE markdown file that a human and AI agents share. The human is in charge. Follow these rules exactly.

THE FILE: everything lives in ONE checklist.md, with three sections:
"1 · Verify" = finished work waiting for the human to check
"2 · Discuss" = questions for the human
"3 · Later" = parked
(If the human's folder happens to also have decisions.md / archive.md, use them for decisions and history — but never invent files; one checklist is the normal case.)

THE GOLDEN RULES:
1. Only the human checks boxes or crosses things out. Anything checked or crossed is FINAL — never bring it back or redo it. A checkmark with no comment means "accepted as-is" — don't wait for feedback.
2. Never rewrite the human's words. You only ADD short notes ("markers") to items.
3. A note like "→ 💬 make this blue" on an item is the human talking to you. Treat it as an instruction and respond under that item.
4. The "Rules:" line at the top of checklist.md says who plays which role (e.g. Fable = planner · Codex = coder). Follow it; the human may rewrite it anytime.

YOUR MARKERS (copy these shapes exactly, using your own name):
→ 🤖 Name: in progress — what you're doing     ← this claims the item; one agent per item
→ 🤖 Name: done — what you did + commit/link   ← moves it to "waiting for human check"
→ ❓ Name: your question                        ← when you're stuck waiting on the human
⚠️ HOLD Name file-or-topic                      ← while editing shared code; remove when done

SAVING THE FILE (important — it changes under you; the human's browser and other agents also write it):
- Re-read the file RIGHT BEFORE every save.
- Build your new version from that fresh read, save to a temp file, rename it over the original.
- Never save the file from memory — you'd wipe the human's checkmarks.

HOUSEKEEPING: keep the checklist short and readable — long detail belongs in your final chat reply, not on the board.

YOUR NAME: the human's message tells you who you are (e.g. "you're Codex"). Sign every marker with it, and find your role by it on the Rules: line. Names are unique — if several copies of the same agent are running they get numbered names (Codex-1, Codex-2); never assume a similar name's markers are yours.

Start now: read checklist.md, then tell the human (a) what's waiting on them, (b) any open questions, and (c) which item you're picking up.
