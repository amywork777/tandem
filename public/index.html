tandem

A shared checklist between you and your AI agents — one markdown file, on your machine, that everyone reads and writes. Agents leave attributed markers; you check the boxes.

Open the board Read the protocol Chromium browsers only (Chrome, Edge, Arc) — the board edits local files.

How it works

  1. Start from notes A meeting transcript, a brain-dump, a plan — anything. Your AI distills it into a checklist file (or click "New checklist" on the board for a blank one).
  2. The file is the board One checklist.md in a project folder, with decisions.md, archive.md, and agents.md beside it. No server — it lives on your machine.
  3. Agents work it under rules Point any agent (Claude, Codex, …) at agents.md. They claim items with in-progress markers, report with done markers, and ask you questions with ❓ — never touching your checkboxes.
  4. You verify in the browser The board re-reads the file every couple of seconds: agent updates just appear. Check what you've verified, cross out what's dead, type feedback right on an item.

Or just paste this into your agent(s)

No setup, no files — copy the block below into any agent (Claude, Codex, anything) and it knows how to play. Works as a system prompt, a first message, or the top of an AGENTS.md.

You're joining a "tandem" board: a checklist in ONE markdown file that a human and AI agents share. The human is in charge. Follow these rules exactly.

THE FILES (all in the project folder):
- checklist.md — the board. Three sections: "1 · Verify" = finished work waiting for the human to check. "2 · Discuss" = questions for the human. "3 · Later" = parked.
- decisions.md — what was decided and why (numbered).
- archive.md — history of finished work.
- agents.md — these rules, project-specific copy.

THE GOLDEN RULES:
1. Only the human checks boxes or crosses things out. Anything checked or crossed is FINAL — never bring it back or redo it. A checkmark with no comment means "accepted as-is" — don't wait for feedback.
2. Never rewrite the human's words. You only ADD short notes ("markers") to items.
3. A note like "→ 💬 make this blue" on an item is the human talking to you. Treat it as an instruction and respond under that item.

YOUR MARKERS (copy these shapes exactly, using your own name):
→ 🤖 Name: in progress — what you're doing     ← this claims the item; one agent per item
→ 🤖 Name: done — what you did + commit/link   ← moves it to "waiting for human check"
→ ❓ Name: your question                        ← when you're stuck waiting on the human
⚠️ HOLD Name file-or-topic                      ← while editing shared code; remove when done

SAVING THE FILE (important — it changes under you; the human's browser and other agents also write it):
- Re-read the file RIGHT BEFORE every save.
- Build your new version from that fresh read, save to a temp file, rename it over the original.
- Never save the file from memory — you'd wipe the human's checkmarks.

HOUSEKEEPING: new decisions (and the why) go in decisions.md. Keep the checklist short — move finished detail to archive.md.

Start now: read checklist.md, then tell the human (a) what's waiting on them, (b) any open questions, and (c) which item you're picking up.
