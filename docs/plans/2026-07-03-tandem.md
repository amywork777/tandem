> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Package Amy's human + AI-pair checklist workflow (local .md SSOT + browser board + coordination protocol) into a hosted, sendable static product with templates and AI bootstrap files.

**Architecture:** Zero-build vanilla static site (`public/` on Vercel). The board is an evolution of the existing `vizcom-docs-drafts/cmf-todo.html` (File System Access API over a local markdown file, 2s poll, line-anchored writes). All rules live once in `PROTOCOL.md` and are mirrored into the help drawer, protocol page, skill, and `templates/agents.md`.

**Tech Stack:** HTML/CSS/vanilla JS, File System Access API (Chromium), Vercel static hosting, Claude Code skill (markdown).

## Global Constraints

- No build step, no framework, no npm dependencies. Each page is self-contained.
- No backend, no analytics, no external requests from pages (fonts/CSS/JS inline or local).
- The markdown file is the single source of truth; the board never reformats or regenerates it — only line-anchored edits of lines the human touched.
- Marker grammar (exact strings, from spec): `→ 🤖 Name: in progress — …` · `→ 🤖 Name: done — …` · `→ ❓ Name: …` · `⚠️ HOLD Name topic`.
- Checkbox semantics: only humans toggle. Board tooltip: "A check means the human tested + verified".
- Visual language: keep the existing board palette (`--bg:#141517 --surface:#1d1e21 --accent:#6650fa --green:#4fc07a --amber:#e0a83c --red:#e06060`).
- Repo: `~/code/tandem`, personal GitHub, deployed with Amy's Vercel token (see memory `reference_vercel_token`).
- Never commit anything from `~/code/vizcom` into this repo.

### Task 2: PROTOCOL.md

**Files:**
- Create: `PROTOCOL.md`

**Interfaces:**
- Produces: canonical rules text. Tasks 3, 4 (help drawer), 5 (protocol.html), 6 (skill, agents.md) quote or summarize THIS file — never invent divergent phrasings.

- [ ] **Step 1: Write PROTOCOL.md** with exactly these sections (prose may be polished, rules must not change):

```markdown
# The tandem protocol

One markdown file is the board. Humans and AI agents all read and write it; these rules keep
that safe and legible.

## Ownership

- **Only humans check boxes or cross items out.** A checked or crossed item is settled: agents
  never resurface, re-open, or re-add it.
- **Agents only append.** An agent responds to an item by appending an attributed marker to that
  item's line (or a new line directly under it) — never by rewriting the human's words.
- Humans may write feedback anywhere, in any form. Inline notes addressed to an agent
  (e.g. "AMY: make the toast clearer") are work orders.

## Markers (exact grammar)

| Marker | Meaning |
|---|---|
| `→ 🤖 Name: in progress — <what>` | Claim + lock. One agent per item. |
| `→ 🤖 Name: done — <what, commit/link>` | Work finished, awaiting human verification. |
| `→ ❓ Name: <question>` | Blocked on the human. Render red; humans scan for these. |
| `⚠️ HOLD Name <file or topic>` | Editing shared code — other agents keep off until the marker is edited to done/removed. |

## The file set

| File | Holds |
|---|---|
| `checklist.md` | Live work only, three sections: **Verify** (done, awaiting human check) · **Discuss** (needs a human/team decision) · **Later** (parked). |
| `decisions.md` | Every decision, numbered, with the WHY. |
| `archive.md` | Append-only history. Move done detail here to keep the checklist short. |
| `agents.md` | Operating instructions for any agent joining the project. Point new agents here first. |

## Division of labor

Architectural or risky items go to the stronger agent; self-contained items to the other.
The in-progress marker IS the claim — no separate assignment step. One agent per item.

## Write safety

The file is hot: the board and other agents write it while you work.

- Re-read the file immediately before every write.
- Write atomically (temp file + rename), replacing the whole file only with content derived
  from that fresh read.
- Never regenerate the file from your memory of it — human checkbox states are sacred.

## Sources in, status out

Meeting notes and transcripts flow IN to the checklist. Outward status (Linear, Notion, Slack)
is generated FROM the checklist when the human asks — never maintained in parallel.
```


`templates/agents.md`:
```markdown
# Agent operating instructions — {{NAME}}

You are one of several agents (plus a human) working `checklist.md` in this folder. Read this
whole file before touching anything.

1. **The checklist is the single source of truth.** Read it before starting; re-read it
   immediately before every write.
2. **Only the human checks boxes or crosses items out.** Settled items are settled — never
   resurface or re-add them.
3. **Append, don't rewrite.** Respond to an item by appending a marker to its line:
   `→ 🤖 <YourName>: in progress — <what>` when you claim it (this IS the lock; one agent per
   item), `→ 🤖 <YourName>: done — <what, commit/link>` when finished,
   `→ ❓ <YourName>: <question>` when blocked on the human.
4. **Shared code:** add `⚠️ HOLD <YourName> <file/topic>` while editing files another agent may
   touch; edit it away when done. Respect others' HOLDs.
5. **Write safety:** the file is hot (a browser board and other agents write it). Write
   atomically (temp file + rename) from a fresh read. Never regenerate the file from memory.
6. **Division of labor:** architectural/risky items → the stronger agent; self-contained →
   the other. If unsure, ask via `→ ❓`.
7. Decisions (with the WHY) go to `decisions.md`, numbered. Finished detail moves to
   `archive.md`. Keep the checklist short.
```

- [ ] **Step 2: Dynamic agent lanes** — replace `renderAgents` hardcoding:

```js
const renderAgents = (lines) => {
  const busy = new Map(); // name -> [items]
  for (const line of lines) {
    const m = line.match(/^\s*- \[[ xX]\]\s*(.*?)\s*→ 🤖 (\S+): in progress/);
    if (m) (busy.get(m[2]) ?? busy.set(m[2], []).get(m[2])).push(
      m[1].replace(/✅|⚠️|🔨|🔴/gu, '').trim()
    );
    for (const h of line.matchAll(/⚠️ HOLD (\S+) ([^·→]*)/g)) {
      (busy.get(h[1]) ?? busy.set(h[1], []).get(h[1])).push('HOLD: ' + h[2].trim());
    }
  }
  const palette = ['claude', 'codex', 'third']; // css classes cycling accent/amber/green
  const lanes = [...busy.entries()].map(([who, items], i) => `
    <div class="lane"><span class="who ${palette[i % palette.length]}">${escapeHtml(who)}</span>
    <span class="what">${escapeHtml(items.join('  ·  '))}</span></div>`);
  // Rule line: first body line starting with "Rules:" (template provides it)
  const rule = lines.find((l) => l.startsWith('Rules:'));
  $('agents').innerHTML = (lanes.join('') || '<div class="lane"><span class="what">no agents in progress</span></div>')
    + (rule ? `<div class="rule">${escapeHtml(rule)}</div>` : '');
  $('agents').hidden = rawMode;
};
```
